# AgentiUX Dev Test Catalog

Canonical e2e planning registry. Each row describes one scenario family and the cleanup contract required for repeatable full-suite execution.

## Executable Suite Map

- `core-full-local`: Wave 1 acceptance suite; remains stable and is not widened by Wave 2 work.
- `wave2-foundation`: Subwave 2A executable coverage.
- `wave2-knowledge`: Subwave 2B executable coverage.
- `wave2-self-host`: Subwave 2C executable coverage.
- `wave2-full-local`: aggregate of every Wave 2 executable case.
- `catalog-implemented-local`: aggregate of `core-full-local` plus `wave2-full-local`.
- `codex-cli-ab-evidence`: supplementary sequential benchmark coverage using real `codex exec` on fresh `bootstrap-assisted` and raw `codex-benchmark-workspace` fixture clones.
- `codex-cli-runtime-warm-evidence`: supplementary single-pass benchmark coverage using real `codex exec` on `runtime-warm` fixture clones after `init-workspace` + `refresh-context-index` and without external bootstrap injection.

## Execution Notes

- `python3 tests/e2e/run.py` still defaults to `core-full-local` unless `--case` or `--suite` is provided.
- Suites that include the fake YouTrack scenarios require localhost bind access for the temporary HTTP server.
- Cheap retrieval working budgets are hard-gated separately from absolute payload ceilings. Stress cases must stay within ceilings even when trimming is required.
- Dashboard performance rows hard-gate request-path invariants plus cold-start, fetch, render, and payload budgets.
